Introdução

A linguagem de programação é um método padronizado, formado por um conjunto de regras sintáticas e semânticas, de implementação de um código fonte - que pode ser compilado e transformado em um programa de computador.



Regras Sintáticas vs Regras Semânticas

Sintáticas = Palavras escritas de forma errada sem mudar o sentido da frase




Regras Semânticas = Palavras escritas de forma certo mas que mudam o sentido da frase



Compiladores e Interpretadores

O compilador gera um arquivo executável que pode ser executado diretamente no sistema operacional. Por outro lado, um interpretador traduz o código fonte linha por linha durante a execução.

Switch

A estrutura de decisão switch permite que o fluxo do programa seja desviado para vários caminhos diferentes a depender do valor passado para análise. Caso o valor analisado não corresponda a nenhuma opção, o bloco de instruções default é executado. Cada bloco de instruções é finalizado com o comando break, que quando executado desvia o programa automaticamente para fora da estrutura do switch.




A estrutura do switch permite que haja um teste lógico para cada case em sua estrutura, funcionando como um conjunto de ifs aninhados.

Programação Orientada à Objeto


No Paradigma de Programação Orientada a Objetos, as Classes são como as formas existentes no mundo das ideias, elas representam os objetos existentes na realidade, mas não são esses objetos propriamente dito, visto que os objetos uma vez instanciados são transitórios, e as classes são na verdade apenas a ideia, a forma dos objetos.







Conceitos Platônicos e Aristotélicos aplicados ao paradigma de Programação Orientada a Objetos

Classes estão inscritas na eternidade, são imutáveis e representam a ideia por traz de um objeto, portanto são genéricas. Para descrever um objeto real, uma classe possui uma definição (um nome) e predicações (atributos e métodos).


O código a baixo mostra a classe que representa objetos do tipo Carro.




Qualquer carro que possua uma cor e uma velocidade (atributos), e que possam acelerar ou frear (métodos) podem ser representados por esta classe.
Vários objetos diferentes podem ser instanciados a partir de uma classe. Objetos estão inscritos na temporalidade, e portanto são transitórios, podendo mudar suas características enquanto existirem.


No exemplo podemos ver diferentes carros sendo instanciados a partir da mesma classe Carro, e seus atributos vão sendo alterados ao longo do programa e deixam de existir quando main() deixar de ser executada.






Estrutura de uma Classe

Classes em programação orientada a objetos são abstrações de objetos, ou seja, representam de forma genérica as instâncias dos objetos. A estrutura de uma classe é divida nome da classe, atributos e métodos.






Encapsulamento

O encapsulamento é um dos princípios fundamentais da programação orientada a objetos, que visa ocultar os detalhes internos de uma classe e expor apenas as interfaces necessárias para interagir com ela. Isso significa que os atributos de uma classe são geralmente declarados como privados, impedindo o acesso direto a eles de fora da classe.

1. Public: Um membro (atributo ou método) marcado como público pode ser acessado de qualquer lugar, ou seja, de dentro da própria classe, de classes do mesmo pacote e de classes de pacotes diferentes.

2. Protected: Um membro protegido pode ser acessado apenas dentro da própria classe, de classes do mesmo pacote e de subclasses (mesmo que estejam em pacotes diferentes).

3. Private: Um membro privado só pode ser acessado dentro da própria classe. Ele não pode ser acessado de fora da classe, nem mesmo por classes do mesmo pacote ou subclasses.




Getters and Setters

O acesso aos atributos de uma classe é fornecido por meio de "métodos públicos", conhecidos como "getters" e "setters".
Esses métodos permitem que os usuários da classe leiam e modifiquem os atributos de maneira controlada, garantindo a consistência e integridade dos dados.






Método Construtor

Em programação orientada a objetos (POO), um método construtor é um tipo especial de método que é chamado automaticamente quando uma instância de uma classe é criada. O objetivo principal de um método construtor é inicializar os atributos de um objeto, garantindo que ele esteja em um estado válido e consistente desde o momento de sua criação.

1. Nome do Método: O nome do método construtor deve ser o mesmo que o nome da classe onde ele está definido. Em Java, por exemplo, o método construtor é declarado usando a palavra-chave public seguida pelo nome da classe.

2. Inicialização de Atributos: O método construtor é usado para inicializar os atributos de um objeto. Isso pode envolver a atribuição de valores iniciais aos atributos da classe ou a execução de outras operações necessárias para configurar o objeto.

3. Execução Automática: O método construtor é chamado automaticamente quando uma nova instância da classe é criada com a palavra-chave new. Isso garante que o objeto seja inicializado corretamente antes de ser usado em outras partes do programa.

4. Construtor Padrão: Se nenhum construtor é explicitamente definido em uma classe, o compilador Java fornece um construtor padrão sem argumentos. Esse construtor padrão simplesmente chama o construtor da superclasse e não faz nenhuma inicialização adicional.




Método toString()

O método toString() é um método especial em muitas linguagens de programação orientadas a objetos, como Java, que é usado para retornar uma representação de string legível para humanos de um objeto. O objetivo principal do método toString() é fornecer uma descrição textual do estado do objeto, geralmente para fins de depuração ou exibição de informações para o usuário.

Implementação Personalizada: Embora a implementação padrão do método toString() na classe Object retorne uma string que consiste no nome da classe seguido do endereço de memória do objeto, é comum e muitas vezes desejável substituir esse comportamento padrão com uma implementação personalizada que forneça informações mais úteis sobre o estado do objeto como por exemplo os valores de seu atributo.

O método toString() é frequentemente usado em situações onde uma representação de string do objeto é necessária, como ao imprimir o objeto no console, ao registrá-lo em logs de depuração ou ao exibi-lo em interfaces de usuário.




Neste código, como estamos substituindo o método toString() da classe Object por uma versão personalizada, precisamos fornecer uma anotação de que o método da superclasse será sobrescrito.

A notação `@Override` é uma anotação em Java que indica explicitamente que um método está sendo sobrescrito da superclasse. Isso significa que o método na subclasse tem o mesmo nome, tipo de retorno e lista de argumentos que um método na superclasse.

Modelando classes no diagrama

Classes são representadas por uma forma retangular, dividida em três partes. A parte superior exibe o nome da classe, enquanto a parte do meio contém os atributos da classe. A parte inferior representa as operações da classe (também conhecida como métodos).




Casos especiais de Associação

Agregação: representa uma relação todo-parte entre o agregado e suas partes. Nessa relação, a perda do todo não resulta, obrigatoriamente, na perda das partes. Sua sintaxe é uma linha com um losango branco na classe que corresponde ao todo.


Composição: também representa uma relação todo-parte entre o agregado e suas parte. Porém a composição representa um forte relacionamento, em que as partes não sobrevivem sem o todo. Sua sintaxe é a de uma linha com um losango preto na classe que corresponde ao todo.




Generalização / Especialização:

Generalização ou Especialização representa a relação existente entre classes. Este relacionamento também é conhecido na orientação a objetos como herança. É quando existe a necessidade de criar uma classe que herde as propriedades de outra classe, isto é, os atributos, métodos e também relacionamentos (associações) da classe superior.

Classes Estáticas

Uma classe estática em programação orientada a objetos é uma classe que não pode ser instanciada e cujos membros (métodos e variáveis) pertencem à classe em si, não a instâncias individuais dela.

Isso significa que você pode acessar os membros estáticos diretamente usando o nome da classe, sem precisar criar um objeto dela.





Sobrecarga de métodos

A sobrecarga de métodos consiste basicamente em criar variações de um mesmo método, ou seja, a criação de dois ou mais métodos com nomes totalmente iguais em uma classe.

A Sobrecarga permite que utilizemos o mesmo nome em mais de um método contanto que suas listas de argumentos sejam diferentes para que seja feita a separação dos mesmos.

Herança

Herança é uma ferramenta simples, porém poderosa, de linguagens orientadas por objetos que permitem que as classes herdem métodos e atributos de outras classes.

Assim com na vida real, no paradigma de programação orientada a objetos, herdar significa receber ou obter algo do seu antecessor (pai).

O benefício principal da herança é a reutilização de código. Herdar métodos de uma superclasse dá à sua classe acesso ao código e aos dados da superclasse. Você não precisará escrever o código duas vezes, o que economiza tempo e otimiza seu código. Além disso, ocorrem poucos erros



Superclasse versus Subclasse

As classes podem derivar ou evoluir de classes pai, o que significa que elas contêm os mesmos métodos e campos que seus pais, mas podem ser consideradas uma forma mais especializada de suas classes pai.

°Superclasses contêm métodos e campos que são passados para todas as suas subclasses

° Subclasses:
- Herdam métodos e campos de suas superclasses
- Podem definir métodos ou campos adicionais que a superclasse não tem



Herdando Construtores

Embora uma subclasse herde todos os métodos e campos de uma classe pai, ela não herda construtores. Você pode criar os construtores da classe filha da forma como preferir. Se você não implementar um construtor para a classe filha, um construtor sem argumento padrão será criado para a classe . No entanto, pode ser útil chamar o construtor da classe pai.







Classes e métodos abstratos

Uma classe abstrata é aquela que não pode ser instanciada:

- Isso significa que não é possível criar objetos desse tipo
- É possível criar variáveis ou referências desse tipo


Por exemplo, a classe Shape é a generalização de todas as classes que representam formas geométricas, mas não faz sentido que a classe Shape seja instanciada, pois ela possui apenas os atributos e métodos que são comuns parar todas formas geométricas, mas não possuí os atributos e métodos necessários para que um retângulo seja desenhado na tela por exemplo.

Para tornar uma classe abstrata utiliza-se a palavra reservada abstract.





Métodos abstratos:

Métodos abstratos:

- Não podem ter um corpo de método
- Devem ser declarados em uma classe abstrata
- Devem ser substituídos em uma subclasse. Isso força os programadores a implementarem e redefinirem métodos Normalmente, as classes abstratas contêm métodos abstratos, métodos parcialmente implementados ou métodos totalmente implementados.

Lembre-se de que as subclasses podem chamar o construtor e os métodos da superclasse usando a palavra-chave super Com classes abstratas, as subclasses também podem usar 'super' para usar o método da superclasse Normalmente, isso é feito primeiro substituindo o método da superclasse, chamando o método super ou substituído e, em seguida, adicionando o código




Polimorfismo

Polimorfismo é um termo que significa "muitas formas". Quando uma variável ou referência pode se referir a diferentes tipos de objetos, isso é chamado de polimorfismo. No caso de programação, o polimorfismo permite que variáveis se refiram a muitos diferentes tipos de objetos, ou seja, elas podem ter várias formas.
