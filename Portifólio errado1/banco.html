Banco de Dados

Introdução

Um banco de dados é uma coleção organizada de informações ou dados estruturados, geralmente armazenados eletronicamente em um sistema de computador. Os bancos de dados são projetados para facilitar o armazenamento, a recuperação e a manipulação eficientes de dados. Eles são usados em uma ampla variedade de aplicações, desde sistemas simples de armazenamento de informações até complexos sistemas de gerenciamento de dados em grandes empresas..


Dados VS Modelagem de Dados

▶ Dados: São informações brutas que podem ser coletadas e armazenadas.

▶ Modelagem de Dados: É o processo de definir a estrutura lógica e as relações dos dados, visando a organização e o acesso eficiente dos mesmos em um sistema de informação.


Estrutura de Dados

● Dados Estruturados
Dados organizados em um formato definido e facilmente acessível por meio de bancos de dados relacionais.

● Dados Não Estruturados
Dados que não têm uma estrutura ou formato predefinido, dificultando sua organização e análise.


● Dados Semi Estruturados
Dados que não são organizados em um formato rígido de tabelas, mas contêm tags ou marcadores que facilitam a organização e a interpretação.




Tipos de Armazenamento

● Privado
Infraestrutura dedicada exclusivamente a uma única organização.

● Publico
Infraestrutura compartilhada entre várias organizações, gerenciada por um provedor de serviços de nuvem.

● Híbrido
Combinação de armazenamento privado e público, permitindo que os dados e aplicativos se movam entre os dois ambientes.


Banco de Dados Relacionais


Modelo de negócio 01:

Em um e-commerce, há duas classes de dados mantidos em um banco de dados. Segue uma breve descrição de cada umas destas classes de dados:

● Entidade Cliente - É quem de fato efetua as compras dentro do sistemas. Seus campos são: código do cliente, primeiro nome, último nome, cpf e endereço.

● Entidade Produto - Armazena os dados de cada produto disponível no estoque do e-commerce. Seus campos são: código do produto, nome do produto, marca, categoria e preço.

● Relacionamento - Estas duas entidades do sistema se relacionam da seguinte forma: Um cliente pode comprar no mínimo 0 produtos ou no máximo N produtos. Já um produto pode ser comprado por no mínimo 0 cliente ou no máximo por 1 cliente.



PRIMARY KEY:
valor único e não ausente que representa cada item salvo no Banco de Dados.

FOREIGN KEY:
é a primary key importada de outra entidade que estabelece relacionamento entre as entidades.

REGRA:
Em um relacionamento 1 para N, a foreign key fica sempre do lado N.


Comandos Banco de Dados


Comandos DDL - Data Definition Language

▶ CREATE TABLE
Cria tabelas

▶ ALTER TABLE
Altera tabelas

▶ DROP TABLE
Deleta tabelas




Comandos DML - Data Manipulation Language

▶ INSERT
Insere dados a tabela

▶ UPDATE
Atualiza linhas

▶ DELETE
Deleta dados da tabela

▶ SELECT
Visualiza toda a tabela




Comandos DQL - Data Query Language

▶ SELECT
Seleciona uma tabela ou coluna em específico

▶ ORDER BY
Exibe a tabela 1 completa com as colunas ordenadas de acordo com a coluna 1 Crescente

▶ JOIN
É utilizado para combinar linhas de duas ou mais tabelas com base em uma condição relacionada entre elas.

▶ GROUP BY
É usado para agrupar registros que possuem o mesmo valor em uma ou mais colunas e aplicar funções agregadas a esses grupos.

▶ UNION
é usado para combinar o resultado de duas ou mais consultas SQL em um único conjunto de resultados

▶ INTERSECT
é usado para retornar as linhas que estão presentes em ambos os conjuntos de resultados de duas consultas SQL

▶ EXCEP
Usado para retornar as linhas que estão presentes no conjunto de resultados da primeira consulta, mas ausentes no conjunto de resultados da segunda consulta

▶ ALIASES
Da apelido as tabelas ou colunas




Comandos DTL - Data Trasaction Language

▶ BEGIN TRANSACTION
É usado para iniciar uma transação no banco de dados. Uma transação é uma sequência de operações de banco de dados que são tratadas como uma unidade indivisível

▶ COMMIT
é usado para confirmar (commitar) as operações de uma transação no banco de dados

▶ ROLLBACK
É usado para reverter (rollback) as operações de uma transação no banco de dados e restaurar o estado anterior ao início da transação
